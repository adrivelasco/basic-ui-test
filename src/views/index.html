CODE CHALLENGE

Hola! Antes de empezar, te recomendaría dedicarle un tiempo a entender la estructura del proyecto.

  • /mock: Contiene dos archivos de formato JSON con la información que posteriormente requerirás para realizar el challenge.

  • /node_modules: Contiene todas las dependencias necesarias para levantar y trabajar sobre el proyecto.

  • /public: Es el output que genera Webpack a partir del código source.

  • /src: Te enfocarás principalmente en este directorio debido a que es el punto entrada y el código base.

EJERCICIOS

1. JQUERY-TEST - DIFICULTAD: BAJA

Mediante jQuery realizar los ejercicios que están en la vista jQuery-Test.

2. 404 NOT FOUND! - DIFICULTAD: MEDIA

Cuando intentamos ingresar a una url inválida (Por ej: "/inexistente") se muestra una vista indicando que hay un 404 ERROR pero para los responsables de Marketing este mensaje no es para nada user-friendly. Por lo cual, crea la tarea para el equipo de Front-End de: Repetir el menú de navegación de la Home; mostrar un mensaje explicándole al usuario de lo que está ocurriendo; y un botón con el texto "Volver a Inicio" que redireccione hacía la Home al presionar sobre él.

Hint: El sitio está hecho son UIKit, podés utilizarlo.

Hint: Ejemplo de como tiene que ser el mensaje de error. Ver imagen

Hint: Para agregar estilos, modificar el archivo "app.scss" ubicado dentro de "basic-ui-test/src/scss".

Hint: Editar sólo el archivo "404.html" que está dentro del directorio views.

3. LISTADO DE POSTS - DIFICULTAD: ALTA

La vista de "Posts" debe mostrar un listado de ellos mostrando: título, imagen, cuerpo del artículo e información del autor del artículo (nombre, email). Para eso realizar una request a "/api/posts" para obtener el listado de posts. Lo mismo para "/api/users" para poder tener los usuarios registrados en la plataforma.

Hint: Editar sólo el archivo "basic-ui-test/src/app.js", el método "posts()".

Hint: Para la HTTP Request podés usar AJAX mediante jQuery o la API nativa del browser, Fetch.

Hint: Una vez que tengas la información tanto de "users" como de "posts". Deberás mediante JS (editar el archivo "app.js"), para iterar sobre el JSON e imprimir en HTML el listado. Te recomiendo leer la documentación de jQuery para saber que métodos utilizar y cómo manipular el DOM.

Hint: En el JSON de "posts", el valor de "author" hace referencia al ID de los usuarios listados en "users". Necesitarás saber esto para mostrar la información del autor en cada artículo.
